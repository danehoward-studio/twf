## Deploy to Vercel

```bash
git init && git add . && git commit -m "deploy"
git remote add origin https://github.com/YOUR/REPO.git
git push -u origin main
```

Vercel → New Project → Import → Framework: **Other** → Deploy

Add environment variable: `ELEVENLABS_API_KEY` → your key → Redeploy

## Audio Caching

Audio is generated by ElevenLabs **once per script** and cached as MP3 in `public/audio/`.
- First play of any voice: calls ElevenLabs, saves MP3, serves it
- All subsequent plays: serves the cached MP3 instantly — no API call

Cached files are committed to GitHub and served as static assets on Vercel.

## Check What Needs Re-encoding

```bash
npm run check-cache
```

This reads `data/athletes.csv`, hashes each `narratorScript` and `photographerScript`,
compares to `data/cache-manifest.json`, and:
- Reports new or changed scripts
- **Deletes** the old MP3 for anything changed (re-encoded on next play)

Run this after editing the CSV before pushing.

## Update Athletes

1. Edit `data/athletes.csv` in Excel or Google Sheets
2. Run `npm run check-cache` to see what audio will be re-generated
3. Save as CSV, replace `data/athletes.csv`
4. Commit and push → Vercel redeploys

### CSV Columns
| Column | Purpose |
|---|---|
| `Status` | `PUBLISHED` = visible |
| `The Athlete Name` | Display name |
| `origin` | e.g. "Atlanta, GA - Defender" |
| `Bio` | Player panel bio |
| `narratorScript` | Text for Narrator voice |
| `photographerScript` | Text for Photographer voice |
| `portrait` | Image URL or `/images/filename.jpg` |

## Change Voice IDs
Edit `api/speak.js` → `VOICE_MAP` at the top.
